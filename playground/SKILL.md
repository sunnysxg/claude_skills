# Playground 实验项目

用户通过 `/playground <描述>` 触发。描述是要验证的问题或要实验的想法。

## 执行步骤

1. **建文件夹**
   - 路径：`playground/{YYYYMMDD}_{有意义的短名}/`
   - 日期取当天（YYYYMMDD，8 位）
   - 名称必须能看出具体做什么，不要用泛泛的词（"测试"、"对比"、"分析"）
   - ❌ 不好：`20260612_dsl对比`、`20260612_因子分析`
   - ✓ 好：`20260612_tsrank_dslparser_vs_calculator`、`20260613_factor_coldstart回填`

2. **写代码**
   - 语言：Python 3，直接用 pandas / numpy / matplotlib 等已装好的库
   - 文件名：**用中文名，描述具体内容**
     - ❌ 不好：`main.py`、`test.py`、`compare.py`
     - ✓ 好：`单股票计算.py`、`两种实现对比.py`、`完整对比报告.ipynb`
   - 代码要能直接跑出结果，不是半成品
   - 不加注释、不写 README、不建 __init__.py
   - 如果需要读数据，在代码里写清楚路径，优先用相对路径或让用户确认

3. **运行验证**
   - 创建完成后立即执行
   - 如果报错，修复后重试
   - 把运行结果展示给用户

## 注意

- 这是一次性实验，不需要工程规范
- 优先让代码跑通、结果直观
- 如果描述模糊，做最合理的假设直接动手，不要反复追问
- 文件名一定要有意义，让人一眼看出是做什么的
